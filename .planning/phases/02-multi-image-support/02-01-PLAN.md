## Recovery procedure (manual use only — NOT part of the autonomous task flow)

If a developer needs to wipe and re-apply the migration locally (for example because they edited the migration.sql by hand and want to test the data-backfill INSERT from a clean slate), the procedure is:

1. Confirm there is no production-relevant data in the local dev database. This wipes ALL local data.
2. Run `pnpm exec prisma migrate reset --force --skip-seed`.
3. Run `pnpm exec prisma migrate dev` to replay every migration (including the appended INSERT).
4. Run `pnpm exec prisma generate`.

This procedure is documented here, OUTSIDE any `<task>` block, so it is never executed automatically. Task 1b's autonomous path uses `--create-only` followed by a normal apply, which is non-destructive: it only adds the new migration to a fresh database state.
